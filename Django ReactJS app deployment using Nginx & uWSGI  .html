Django React app deployment using Nginx + uWSGI
Step: 1 Create a Django and ReactJS applications.
Here is the sample code : https://github.com/kom3/REACTJS_DJANGO_Nginx_uWSGI_DEPLOYMENT.git

Make sure you have build files/static files ready before getting start with deployment:
For react use : npm run build
For Django(to collect static files) use: python3 manage.py collectstatic
Step: 2 Setting up the uWSGI application server.
Install uWSGI library:
In ubuntu:
sudo apt-get install python3-dev
sudo -H pip3 install uwsgi
Step: 3 Creating configuration files.
sudo mkdir -p /etc/uwsgi/sites
sudo vim /etc/uwsgi/sites/djangoreact.ini


Paste the following content:


[uwsgi]
project = djangoserver
#user
uid = ko
#group
gid = www-data
#project base dir(below used as a variable)
base = /home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT/djangoserver

chdir = %(base)
#virtual env path
home = /home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT/nginxuwsgi
#django wsgi file path
module = %(project).wsgi:application

master = true processes = 33 buffer-size=32768
#master = true
processes = 5

#using socket for communicating, this works only for linux not windows (can also use http proxy)
#path where uWSGI has to create a .sock file
socket = %(base)/%(project)/%(project).sock
#to check uWSGI reachability without nginx use http (for debugging uWSGI)
#http = 127.0.0.1:8091
chmod-socket = 660
vacuum = true
Step: 4 Creating systemd unit file for uWSGI (we can make it run as Linux service).
sudo nano /etc/systemd/system/uwsgi.service


[Unit]
Description=uWSGI Emperor service

[Service]
#setting required permissions for project folder
ExecStartPre=/bin/bash -c 'mkdir -p /home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT; chown ko:www-data /home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT'
ExecStart=/usr/local/bin/uwsgi --emperor /etc/uwsgi/sites
Restart=always
KillSignal=SIGQUIT
Type=notify
NotifyAccess=all


[Install]
WantedBy=multi-user.target
Step: 5 Installing and configuring Nginx as a reverse proxy (requests hit the Nginx and Nginx will forward those requests to uWSGI server through socket).
sudo apt-get install nginx
sudo vim /etc/nginx/sites-available/djangoreact

paste the following content:

server {
    listen 8090;
    server_name djangoreact.com www.djangoreact.com;

    location = /home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT/Reactapp/myapp/build/favicon.ico { access_log off; log_not_found off; }
    location / {
        root /home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT/Reactapp/myapp/build;
    }

    location /api/ {
alias /;
        include         uwsgi_params;
        uwsgi_pass      unix:/home/ko/REACTJS_DJANGO_APACHE_DEPLOYMENT/djangoserver/djangoserver/djangoserver.sock;
    }
}
Step: 6 Create a soft link from  /etc/nginx/sites-available/djangoreact to /etc/nginx/sites-enabled/ and restart Nginx server.
sudo ln -s /etc/nginx/sites-available/djangoreact /etc/nginx/sites-enabled

check configuration syntax by typing:
sudo nginx -t

Restart Nginx by typing:
sudo systemctl restart nginx

Restart uWSGI server by typing:
sudo systemctl start uwsgi
If site is not reachable just check if the ports are allowed or not
